## Feature 019: Prisma Cloud Scan Import

ATO Copilot now integrates with **Palo Alto Prisma Cloud** to import cloud posture findings, map them to NIST 800-53 controls, and track remediation trends over time. This feature adds 4 new MCP tools, 2 parsers, a severity mapper, and full downstream pipeline integration.

---

### New MCP Tools

| Tool | Description | RBAC |
|------|-------------|------|
| `compliance_import_prisma_csv` | Import Prisma Cloud console CSV exports with severity mapping, NIST control resolution, and dry-run preview | ISSM, ISSO |
| `compliance_import_prisma_api` | Import Prisma Cloud API v2 alert JSON with full resource metadata, CLI remediation scripts, and auto-resolve subscription support | ISSM, ISSO |
| `compliance_list_prisma_policies` | List Prisma policies from imported data with filtering by severity, cloud type, compliance standard, and enabled status | All compliance roles |
| `compliance_prisma_trend` | Analyze posture trends across imports with configurable time windows and grouping (severity, policy, resource type, region, compliance standard) | All compliance roles |

---

### Key Capabilities

#### CSV Import (`compliance_import_prisma_csv`)
- Parse Prisma Cloud console CSV exports with automatic column-header detection
- Map Prisma severity levels (Critical/High/Medium/Low/Informational) to RMF-aligned finding categories
- Resolve Prisma policy IDs to NIST 800-53 controls via CCI crosswalk
- Dry-run mode previews import results without persisting data
- Multi-subscription resolution matches Prisma accounts to registered systems

#### API JSON Import (`compliance_import_prisma_api`)
- Parse Prisma Cloud API v2 alert responses with nested resource metadata
- Extract CLI remediation scripts (e.g., `az storage account update ...`) for engineer use
- Capture compliance standard mappings from alert details
- Auto-resolve subscription mode to link Prisma subscriptions to ATO Copilot systems
- Structured logging with warnings for unresolved subscriptions

#### Prisma Policy Catalog (`compliance_list_prisma_policies`)
- Query policies by severity, cloud type, compliance standard, and enabled status
- View policy-to-NIST control mappings and per-policy alert counts
- Identify unmapped policies for manual CCI crosswalk

#### Trend Analysis (`compliance_prisma_trend`)
- Compare scan imports across configurable time windows
- Group by severity, policy, resource type, region, or compliance standard
- Track open/resolved/new alert counts per period for ConMon drift detection
- Supports monthly and quarterly cadence aligned to ConMon plans

### Downstream Integration

Prisma imports feed into the existing RMF pipeline — no changes needed to downstream tools:

| Downstream Tool | Integration |
|-----------------|-------------|
| **SAR Generation** (`compliance_generate_sar`) | Prisma findings contribute to ControlEffectiveness records used in SAR |
| **POA&M** (`compliance_create_poam`) | Dismissed Prisma findings map to Accepted status for POA&M consideration |
| **ConMon Report** (`compliance_generate_conmon_report`) | Prisma open/resolved findings included in continuous monitoring reports |
| **RAR Generation** | Findings queryable by assessment for Risk Assessment Report |
| **Authorization Package** | SHA-256 hashed evidence records created for each import |
| **Dashboard** (`compliance_multi_system_dashboard`) | Prisma findings scoped per-system for multi-system posture views |
